## 2024-03-05 - Add tooltips to topbar icon buttons
**Learning:** Radix tooltips require the `<TooltipProvider>` to be wrapped around a parent component, ideally globally. The `apps/web/components/providers.tsx` file is the correct place to inject global context wrappers like `TooltipProvider` in this architecture. Also, shadcn/ui buttons generally require explicit `focus-visible` styling (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary/20`) to ensure proper keyboard navigation visibility if they are raw `<button>` elements instead of the `Button` component.
**Action:** Always check if a global provider is needed when implementing new Radix UI components (like tooltips or toasts). When adding accessible tooltips to raw buttons, ensure custom `focus-visible` ring styles are applied since they don't inherit the default `Button` component styles.
